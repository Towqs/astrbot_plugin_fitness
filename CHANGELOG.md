# 更新日志

## v2.0.3

### Bug 修复
- 修复打卡提醒重复发送：添加并发锁 + `max_instances=1` + 提前标记已提醒，防止同一用户被重复提醒
- 修复数据泄漏：系统提示中明确禁止 AI 直接展示工具返回的 JSON 原始数据，必须用自然语言转述

### 新功能
- 建档完成后自动生成4周训练周期计划，用户无需手动触发

## v2.0.2

### 新功能
- 主动回复：教练以可配置概率主动回复群聊消息，概率可在配置中调整（默认关闭，建议5-15%）

## v2.0.1

### Bug 修复
- 修复 `update_status` 更新其他字段时会意外清除补剂状态的问题
- 修复 `create_profile` 分步建档时补剂状态被覆盖的问题
- 修复更新体重时未写入体重记录表，导致进步报告体重趋势为空
- 修复补卡不推进闯关进度的问题
- 修复补卡不触发成就检查的问题
- 修复饮食打卡未检查用户是否已建档的问题
- 修复成就经验导致升级时不更新群头衔的问题
- 修复训练周期 `current_week` 永远停在第1周不自动推进的问题
- 修正 README 中补卡描述与实际行为不一致（仅支持补前一天）

## v2.0.0

### 新功能
- 周期化训练（4-8周，渐进超负荷 + 去负荷周）
- 疲劳度评估（打卡时自动评估，高疲劳触发建议）
- 饮食打卡（自然语言记录，AI 估算热量和蛋白质）
- 成就系统（12种成就，多种触发条件）
- 进步检测（自动对比近期与历史数据）
- 用户画像（训练偏好、恢复能力、疲劳趋势）
- 群周报（每周日自动发送排行榜和统计）
- 周六体重反馈（定时询问体重变化）
- 补卡机制（补前一天的卡，经验减半）
- 渐进式信息收集（日常对话中自然收集用户信息）
- AI 综合分析（自动生成训练方向建议）
- 双模型支持（高成本对话 + 低成本提醒）
- 5种教练人格可选
